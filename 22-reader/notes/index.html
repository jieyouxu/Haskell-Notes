    Reader¶

    Sometimes "globals" or environmental configurations need to be passed around the entire application. But such information should not be passed via arguments since it will clutter the types of every function. The Reader datatype can be used to handle this burden.

    Functorial Context

    Normally, function composition is done with the composition operator (.).

    fn_a :: Integer -> Integer
fn_a = (* 2)

fn_b :: Integer -> Integer
fn_b = (+ 4)

fn_c :: Integer -> Integer
fn_c = fn_a . fn_b

    Upon applying fn_c to some input, fn_b will be applied to the input first, whose result will be passed on to fn_a.

    Function composition can also be achieved via fmap. Here, the structure or context for fn_c is a partially applied function.

    Functor Instance for Function

    The Functor instance for a partially applied function (from a function of type a -> r, or (->) a r) is:

    instance Functor ((->) a) where
    -- fmap :: (b -> c) -> ((->) a) b -> ((->) a) c
    -- fmap f g = \x -> f (g x)
    fmap = (.)

    Applicative Context

    fn_d :: Integer -> Integer
fn_d = (+) <$> fn_a <*> fn_b

-- or equivalently
fn_d' :: Integer -> Integer
fn_d' = liftA2 (+) fn_a fn_b

    In this case the argument will get passed to fn_a and fn_b simultaneously and their results will be added together.

    Applicative Instance for Function

    instance Applicative ((->) r) where
    -- (<*>) :: ((->) r) (a -> b) -> ((->) r) a -> ((->) r) b
    -- f :: r -> a -> b
    -- g :: r -> a
    pure x  = \_ -> x
    f <*> g = \x -> f x (g x)

    Monad Instance for Function

    instance Applicative ((->) r) => Monad ((->) r) where
    -- return :: a -> r -> a
    return :: a -> ((->) r) a
    return = const

    -- (>>=) :: (r -> a) -> (a -> r -> b) -> (r -> b)
    (>>=) :: ((->) r) a -> (a -> ((->) r) b) -> ((->) r) b
    -- x :: r
    -- f :: r -> a
    -- g :: a -> r -> b
    -- f >>= g = \x -> g (f x) x
    f >>= g = flip g <*> f

    Reader

    Partially applied functions have Functor, Applicative as well as Monad instance.

    • Functor for functions is composition.
    • Applicative and Monad instance pass argument forward in addition to the composition.
      • Notice the input argument r is present in the created result function (r -> b).
      • Recall that Applicative and Monad are subclasses of Functor.

    The Reader newtype is an abstraction for composing together functions which are waiting for shared environmental inputs – it is an abstraction of function application and allows computation to be done in terms of some not-yet-supplied argument. Reader can help us avoid passing some environmental information explicitly.

    newtype Reader r a = Reader { runReader :: r -> a }

    r is the input type and a is the result type of the function. The newtype defines a convenient accessor function runReader which allows the function to be retrieved from the newtype. Reader r is similar to ((->) r).

    instance Functor (Reader r) where
    fmap :: (a -> b) -> Reader r a -> Reader r b
    -- fmap f (Reader ra) = Reader $ \r -> f (ra r)
    fmap f (Reader ra) = Reader $ (f . ra)

    Notice the similarity between \r -> f (ra r) and composition \x -> f (g x). Thus we can use (.) to simplify.

    Read-only Type Argument¶

    The r type argument being read-only imposes the restriction that r can only be swapped for your callees but not for your callers.

    -- swap Reader context
withReaderT :: (r' -> r) -> ReaderT r m a -> ReaderT r' m a
withReaderT f m = ReaderT $ runReaderT m . f
